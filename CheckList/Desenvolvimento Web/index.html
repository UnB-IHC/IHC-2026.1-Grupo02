    Desenvolvimento Web

    Imagem

    • Imagens informativas, gráficos, mapas e botões-imagem possuem texto alternativo descritivo. [1]
    • Imagens decorativas utilizam alt="" para serem ignoradas por leitores de tela. [1]
    • Informações apresentadas como texto em imagens também estão disponíveis em formato textual. [1]
    • Textos presentes em imagens possuem contraste adequado com o plano de fundo. [8]
    • Informações importantes não dependem exclusivamente de texto incorporado em imagens. [1]
    • Imagens complexas possuem descrição complementar quando necessário. [1]

    Vídeo

    • Vídeos possuem legendas sincronizadas para o conteúdo falado. [3]
    • Conteúdos apenas em áudio possuem transcrição textual equivalente. [2]
    • Vídeos e animações evitam flashes acima dos limites recomendados. [13]
    • Informações importantes do vídeo estão disponíveis para usuários que não podem vê-lo ou ouvi-lo. [2][3]
    • Áudios reproduzidos automaticamente podem ser pausados ou interrompidos pelo usuário. [7]

    Controles

    • Links possuem destino válido e funcional. [4]
    • Links possuem indicação visual além da mudança de cor. [6]
    • Links que abrem novas abas ou saem do sistema são identificados ao usuário. [20]
    • O texto dos links descreve claramente seu destino ou finalidade. [16]
    • Botões possuem texto ou descrição que identifica sua função. [4]
    • Elementos interativos apresentam indicação visual de foco. [18]
    • A navegação por foco segue uma ordem lógica. [15]
    • Existe contraste adequado nos elementos interativos. [10]
    • Existe um atalho para acessar diretamente o conteúdo principal da página. [14]

    Formulário

    • Cada campo possui um rótulo associado e claramente identificável. [25]
    • Grupos de opções utilizam fieldset e legend quando apropriado. [4]
    • Campos utilizam autocomplete quando aplicável. [5]
    • Campos obrigatórios são identificados antes do envio. [25]
    • Mensagens de erro explicam claramente como corrigir o problema. [26]
    • Instruções e mensagens de erro estão associadas ao campo correspondente. [24][25]
    • Validações em tempo real são acessíveis para tecnologias assistivas. [23]
    • Os erros são identificados de forma clara após o envio do formulário. [24]
    • O preenchimento dos campos não provoca mudanças inesperadas na interface. [23]

    Semântica

    • A página utiliza elementos HTML semânticos adequados (header, nav, main, footer, etc.). [4]
    • A estrutura do conteúdo segue uma organização lógica. [4]
    • Tabelas são utilizadas apenas para apresentação de dados. [4]
    • Trechos em outros idiomas possuem identificação apropriada. [22]
    • Componentes e regiões da página possuem significado claro para tecnologias assistivas. [4]

    Texto

    • O uso de texto dentro de imagens é evitado sempre que possível. [1]
    • O conteúdo permanece legível com zoom de até 200%. [9]
    • Fontes utilizam dimensionamento flexível sempre que possível. [9]
    • Os textos utilizam linguagem clara e objetiva. [27][28]
    • Informações não dependem exclusivamente de cor para serem compreendidas. [6]
    • Rótulos, instruções e mensagens são fáceis de entender. [17][25]
    • O contraste entre texto e plano de fundo é adequado. [8]

    Teclado

    • Todas as funcionalidades podem ser utilizadas apenas com o teclado. [11]
    • Componentes acionados por mouse também podem ser utilizados por teclado. [11]
    • O foco do teclado pode percorrer todos os elementos interativos. [11]
    • O foco permanece visível durante toda a navegação. [18]
    • Não existem armadilhas de foco que impeçam a navegação. [11]
    • O primeiro elemento interativo permite acessar diretamente o conteúdo principal. [14]
    • A sequência de navegação acompanha a ordem lógica da interface. [15]
    • Atalhos de teclado não interferem em tecnologias assistivas. [28]

    Títulos

    • A página possui um título principal representado por um elemento h1. [17]
    • Os títulos seguem uma hierarquia lógica (h1, h2, h3...). [17]
    • Não existem saltos desnecessários entre níveis de títulos. [17]
    • Os títulos descrevem corretamente o conteúdo da seção. [17]

    Tabelas

    • Tabelas utilizam o elemento table. [4]
    • Cabeçalhos utilizam o elemento th quando apropriado. [4]
    • O atributo scope é utilizado para associar cabeçalhos e células. [4]
    • Tabelas possuem legenda descritiva utilizando caption. [4]
    • Tabelas não são utilizadas para criar layouts visuais. [4]

    Modais

    • O foco é direcionado para o modal quando ele é aberto. [11][18]
    • O foco retorna ao elemento anterior quando o modal é fechado. [11][18]
    • O modal pode ser fechado utilizando a tecla ESC. [11]
    • Existe uma forma clara de fechar o modal. [27][28]
    • Não existem modais abertos dentro de outros modais. [27][28]

    Dispositivos móveis e toque

    • A interface funciona corretamente em diferentes orientações da tela. [29]
    • Não existe rolagem horizontal desnecessária. [9]
    • Elementos interativos possuem tamanho adequado para toque. [19]
    • Existe espaçamento suficiente entre elementos clicáveis. [19]
    • Gestos de toque possuem alternativas acessíveis. [28]

    Ferramentas e extras

    • Conteúdos em movimento podem ser pausados, interrompidos ou ocultados. [12]
    • O idioma principal da página está definido na tag html. [21]
    • Existem orientações ou recursos de acessibilidade quando necessário. [27][28]
    • CAPTCHAs possuem alternativas acessíveis. [27][28]
    • Existe mecanismo de busca quando necessário para facilitar a localização de conteúdo. [27]
    • Áreas clicáveis possuem tamanho adequado para interação. [19]
