## Imagem

* <input type="checkbox" data-id="dev-imagem-1"> Imagens informativas, gráficos, mapas e botões-imagem possuem texto alternativo descritivo. [1]
* <input type="checkbox" data-id="dev-imagem-2"> Imagens decorativas utilizam alt="" para serem ignoradas por leitores de tela. [1]
* <input type="checkbox" data-id="dev-imagem-3"> Informações apresentadas como texto em imagens também estão disponíveis em formato textual. [1]
* <input type="checkbox" data-id="dev-imagem-4"> Textos presentes em imagens possuem contraste adequado com o plano de fundo. [8]
* <input type="checkbox" data-id="dev-imagem-5"> Informações importantes não dependem exclusivamente de texto incorporado em imagens. [1]
* <input type="checkbox" data-id="dev-imagem-6"> Imagens complexas possuem descrição complementar quando necessário. [1]

## Vídeo

* <input type="checkbox" data-id="dev-video-1"> Vídeos possuem legendas sincronizadas para o conteúdo falado. [3]
* <input type="checkbox" data-id="dev-video-2"> Conteúdos apenas em áudio possuem transcrição textual equivalente. [2]
* <input type="checkbox" data-id="dev-video-3"> Vídeos e animações evitam flashes acima dos limites recomendados. [13]
* <input type="checkbox" data-id="dev-video-4"> Informações importantes do vídeo estão disponíveis para usuários que não podem vê-lo ou ouvi-lo. [2][3]
* <input type="checkbox" data-id="dev-video-5"> Áudios reproduzidos automaticamente podem ser pausados ou interrompidos pelo usuário. [7]

## Controles

* <input type="checkbox" data-id="dev-controles-1"> Links possuem destino válido e funcional. [4]
* <input type="checkbox" data-id="dev-controles-2"> Links possuem indicação visual além da mudança de cor. [6]
* <input type="checkbox" data-id="dev-controles-3"> Links que abrem novas abas ou saem do sistema são identificados ao usuário. [20]
* <input type="checkbox" data-id="dev-controles-4"> O texto dos links descreve claramente seu destino ou finalidade. [16]
* <input type="checkbox" data-id="dev-controles-5"> Botões possuem texto ou descrição que identifica sua função. [4]
* <input type="checkbox" data-id="dev-controles-6"> Elementos interativos apresentam indicação visual de foco. [18]
* <input type="checkbox" data-id="dev-controles-7"> A navegação por foco segue uma ordem lógica. [15]
* <input type="checkbox" data-id="dev-controles-8"> Existe contraste adequado nos elementos interativos. [10]
* <input type="checkbox" data-id="dev-controles-9"> Existe um atalho para acessar diretamente o conteúdo principal da página. [14]

## Formulário

* <input type="checkbox" data-id="dev-formulario-1"> Cada campo possui um rótulo associado e claramente identificável. [25]
* <input type="checkbox" data-id="dev-formulario-2"> Grupos de opções utilizam fieldset e legend quando apropriado. [4]
* <input type="checkbox" data-id="dev-formulario-3"> Campos utilizam autocomplete quando aplicável. [5]
* <input type="checkbox" data-id="dev-formulario-4"> Campos obrigatórios são identificados antes do envio. [25]
* <input type="checkbox" data-id="dev-formulario-5"> Mensagens de erro explicam claramente como corrigir o problema. [26]
* <input type="checkbox" data-id="dev-formulario-6"> Instruções e mensagens de erro estão associadas ao campo correspondente. [24][25]
* <input type="checkbox" data-id="dev-formulario-7"> Validações em tempo real são acessíveis para tecnologias assistivas. [23]
* <input type="checkbox" data-id="dev-formulario-8"> Os erros são identificados de forma clara após o envio do formulário. [24]
* <input type="checkbox" data-id="dev-formulario-9"> O preenchimento dos campos não provoca mudanças inesperadas na interface. [23]

## Semântica

* <input type="checkbox" data-id="dev-semantica-1"> A página utiliza elementos HTML semânticos adequados (header, nav, main, footer, etc.). [4]
* <input type="checkbox" data-id="dev-semantica-2"> A estrutura do conteúdo segue uma organização lógica. [4]
* <input type="checkbox" data-id="dev-semantica-3"> Tabelas são utilizadas apenas para apresentação de dados. [4]
* <input type="checkbox" data-id="dev-semantica-4"> Trechos em outros idiomas possuem identificação apropriada. [22]
* <input type="checkbox" data-id="dev-semantica-5"> Componentes e regiões da página possuem significado claro para tecnologias assistivas. [4]

## Texto

* <input type="checkbox" data-id="dev-texto-1"> O uso de texto dentro de imagens é evitado sempre que possível. [1]
* <input type="checkbox" data-id="dev-texto-2"> O conteúdo permanece legível com zoom de até 200%. [9]
* <input type="checkbox" data-id="dev-texto-3"> Fontes utilizam dimensionamento flexível sempre que possível. [9]
* <input type="checkbox" data-id="dev-texto-4"> Os textos utilizam linguagem clara e objetiva. [27][28]
* <input type="checkbox" data-id="dev-texto-5"> Informações não dependem exclusivamente de cor para serem compreendidas. [6]
* <input type="checkbox" data-id="dev-texto-6"> Rótulos, instruções e mensagens são fáceis de entender. [17][25]
* <input type="checkbox" data-id="dev-texto-7"> O contraste entre texto e plano de fundo é adequado. [8]

## Teclado

* <input type="checkbox" data-id="dev-teclado-1"> Todas as funcionalidades podem ser utilizadas apenas com o teclado. [11]
* <input type="checkbox" data-id="dev-teclado-2"> Componentes acionados por mouse também podem ser utilizados por teclado. [11]
* <input type="checkbox" data-id="dev-teclado-3"> O foco do teclado pode percorrer todos os elementos interativos. [11]
* <input type="checkbox" data-id="dev-teclado-4"> O foco permanece visível durante toda a navegação. [18]
* <input type="checkbox" data-id="dev-teclado-5"> Não existem armadilhas de foco que impeçam a navegação. [11]
* <input type="checkbox" data-id="dev-teclado-6"> O primeiro elemento interativo permite acessar diretamente o conteúdo principal. [14]
* <input type="checkbox" data-id="dev-teclado-7"> A sequência de navegação acompanha a ordem lógica da interface. [15]
* <input type="checkbox" data-id="dev-teclado-8"> Atalhos de teclado não interferem em tecnologias assistivas. [28]

## Títulos

* <input type="checkbox" data-id="dev-titulo-1"> A página possui um título principal representado por um elemento h1. [17]
* <input type="checkbox" data-id="dev-titulo-2"> Os títulos seguem uma hierarquia lógica (h1, h2, h3...). [17]
* <input type="checkbox" data-id="dev-titulo-3"> Não existem saltos desnecessários entre níveis de títulos. [17]
* <input type="checkbox" data-id="dev-titulo-4"> Os títulos descrevem corretamente o conteúdo da seção. [17]

## Tabelas

* <input type="checkbox" data-id="dev-tabela-1"> Tabelas utilizam o elemento table. [4]
* <input type="checkbox" data-id="dev-tabela-2"> Cabeçalhos utilizam o elemento th quando apropriado. [4]
* <input type="checkbox" data-id="dev-tabela-3"> O atributo scope é utilizado para associar cabeçalhos e células. [4]
* <input type="checkbox" data-id="dev-tabela-4"> Tabelas possuem legenda descritiva utilizando caption. [4]
* <input type="checkbox" data-id="dev-tabela-5"> Tabelas não são utilizadas para criar layouts visuais. [4]

## Modais

* <input type="checkbox" data-id="dev-modal-1"> O foco é direcionado para o modal quando ele é aberto. [11][18]
* <input type="checkbox" data-id="dev-modal-2"> O foco retorna ao elemento anterior quando o modal é fechado. [11][18]
* <input type="checkbox" data-id="dev-modal-3"> O modal pode ser fechado utilizando a tecla ESC. [11]
* <input type="checkbox" data-id="dev-modal-4"> Existe uma forma clara de fechar o modal. [27][28]
* <input type="checkbox" data-id="dev-modal-5"> Não existem modais abertos dentro de outros modais. [27][28]

## Dispositivos móveis e toque

* <input type="checkbox" data-id="dev-mobile-1"> A interface funciona corretamente em diferentes orientações da tela. [29]
* <input type="checkbox" data-id="dev-mobile-2"> Não existe rolagem horizontal desnecessária. [9]
* <input type="checkbox" data-id="dev-mobile-3"> Elementos interativos possuem tamanho adequado para toque. [19]
* <input type="checkbox" data-id="dev-mobile-4"> Existe espaçamento suficiente entre elementos clicáveis. [19]
* <input type="checkbox" data-id="dev-mobile-5"> Gestos de toque possuem alternativas acessíveis. [28]

## Ferramentas e extras

* <input type="checkbox" data-id="dev-extra-1"> Conteúdos em movimento podem ser pausados, interrompidos ou ocultados. [12]
* <input type="checkbox" data-id="dev-extra-2"> O idioma principal da página está definido na tag html. [21]
* <input type="checkbox" data-id="dev-extra-3"> Existem orientações ou recursos de acessibilidade quando necessário. [27][28]
* <input type="checkbox" data-id="dev-extra-4"> CAPTCHAs possuem alternativas acessíveis. [27][28]
* <input type="checkbox" data-id="dev-extra-5"> Existe mecanismo de busca quando necessário para facilitar a localização de conteúdo. [27]
